# zk-proof-of-origin

Cryptographic proof of content origin on Mina Protocol. One zkProgram, one proof: "this content was made by a verified human" or "this content was generated by AI model X."

Built entirely on Mina's native primitives: recursive zk-SNARKs, httpz (web proofs), and zkML.

## Why this exists

Creators need a way to label their work as authentically human-made or transparently AI-generated. Current solutions rely on platform trust, watermarks that can be stripped, or centralized AI detectors that guess. This project replaces all of that with math: a single succinct proof that anyone can verify in under a second, with no trust in any platform and no data leaked.

## Architecture

**Monorepo with two packages:**

- `packages/circuits` — o1js zkProgram (recursive proof circuit, deployable as Mina zkApp)
- `packages/web` — Browser-based creator tool (React/Vite, client-side proof generation)

**Phase plan:**

| Phase | Scope | Status |
|-------|-------|--------|
| 0 | Repo scaffold, o1js hello-world circuit, tooling | Current |
| 1 | Human provenance (httpz credential proof + content hash) | Next |
| 2 | AI branch (zkML model inference proof) | Future |
| 3 | On-chain zkApp deployment + verifier UI | Future |

See [DECISIONS.md](DECISIONS.md) for architectural rationale and [docs/architecture.md](docs/architecture.md) for system design.

## Getting started

```bash
# Install dependencies (requires Node.js >= 18)
npm install

# Run circuit tests
cd packages/circuits && npm test

# Start web dev server
cd packages/web && npm run dev
```
